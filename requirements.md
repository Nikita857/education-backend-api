# Техническое задание: Функционал корпоративного портала обучения

## Введение

**Цель:** Создать централизованную, удобную и эффективную платформу для управления знаниями, обучения и развития сотрудников компании.

**Ключевые ценности:**
*   Повышение квалификации сотрудников.
*   Стандартизация и контроль качества обучения.
*   Упрощение адаптации новых сотрудников.
*   Снижение издержек на обучение.
*   Создание культуры непрерывного обучения.
*   Управление компетенциями и карьерным ростом.

---

## 1. Пользовательские роли и права доступа

| Роль | Описание | Ключевые права |
| :--- | :--- | :--- |
| **Сотрудник (Learner)** | Основной пользователь, проходит обучение. | Просмотр каталога курсов, запись на курсы, прохождение материалов, отслеживание прогресса, получение сертификатов. |
| **Менеджер (Manager)** | Руководитель отдела. | Просмотр прогресса и отчетов по своему отделу, назначение обучения сотрудникам, формирование ИПР (Индивидуального плана развития). |
| **Преподаватель/Тьютор (Instructor)** | Создает контент и проводит обучение. | Создание и редактирование курсов, проверка заданий, ведение вебинаров, общение с учениками. |
| **Администратор (Admin)** | Полный контроль над системой. | Управление пользователями, ролями, каталогом курсов, настройками системы, генерация глобальных отчетов. |
| **HR / L&D менеджер** | Ответственный за развитие персонала. | Аналитика по обучению, управление программами адаптации, создание учебных программ, работа с кадровым резервом. |

---

## 2. Основные функциональные модули

### 2.1. Модуль управления пользователями и ролями
*   **Регистрация и профиль:**
    *   Интеграция с Active Directory / корпоративной почтой для единого входа (SSO).
    *   Личный кабинет с фото, должностью, навыками, историей обучения.
    *   Возможность указывать интересы и цели развития.
*   **Группы и отделы:** Организация пользователей по структурному принципу компании.
*   **Гибкая система ролей:** Настраиваемые права доступа для разных ролей.

### 2.2. Модуль каталога курсов и контента
*   **Публичный каталог:** Удобный поиск и фильтрация курсов по категориям, тегам, уровню сложности, формату.
*   **Форматы контента:**
    *   **SCORM/xAPI (Tin Can):** Поддержка стандартов для интерактивных курсов.
    *   **Видеолекции:** С поддержкой различных форматов, плейлистов и контроля просмотра.
    *   **Текстовые материалы:** PDF, PowerPoint, Word.
    *   **Интерактивные элементы:** Тесты, опросы, симуляции.
    *   **Вебинары:** Интеграция с Zoom, MS Teams, etc. (запись, автозапись участников).
*   **Система рейтингов и отзывов:** Возможность оценивать курсы и оставлять комментарии.

### 2.3. Модуль обучения и прогресса
*   **Личный учебный план (ИПР):** Индивидуальный набор курсов, назначенных руководителем или выбранных сотрудником.
*   **Запись на курсы:** Свободная или по approval-потоку (согласование с руководителем).
*   **Трекер прогресса:** Наглядное отображение статуса ("Не начат", "В процессе", "Завершен"), процента прохождения.
*   **Система напоминаний:** Уведомления о дедлайнах, новых курсах, незавершенном обучении.
*   **Электронный сертификат:** Автоматическая генерация сертификатов при успешном завершении курса.

### 2.4. Модуль тестирования и оценки
*   **Конструктор тестов:** Различные типы вопросов (один/несколько правильных ответов, текстовый ответ, сопоставление).
*   **Настройка прохождения:** Порог прохождения, ограничение по времени, количество попыток.
*   **Рандомизация вопросов:** Для повышения объективности оценки.
*   **Обратная связь:** Показ правильных ответов после прохождения (по настройкам).

### 2.6. Модуль отчетности и аналитики
*   **Предустановленные отчеты:**
    *   Отчет по сотруднику: вся история и эффективность обучения.
    *   Отчет по курсу: статистика прохождения, успеваемости, вовлеченности.
    *   Отчет по отделу: процент обученности, выполнение планов.
*   **Конструктор отчетов:** Возможность создавать кастомные отчеты с выбором полей и фильтров.
*   **Визуализация данных:** Графики и дашборды для руководства.
*   **Экспорт данных:** Выгрузка отчетов в Excel/PDF.

### 2.7. Модуль адаптации новых сотрудников
*   **Конструктор программ адаптации:** Создание пошаговых планов для разных должностей.
    *   *Задача 1:* Познакомиться с руководителем.
    *   *Задача 2:* Пройти курс по продукту.
    *   *Задача 3:* Оформить документы в HR.
*   **Назначение ментора/наставника.**
*   **Трекер выполнения:** Руководитель и HR видят прогресс адаптации.

### 2.8. Модуль управления навыками (Skills Management)
*   **Матрица компетенций:** Словарь навыков для каждой должности.
*   **Оценка навыков:** Самооценка сотрудника, оценка руководителем, проверка через тестирование.
*   **Выявление разрывов (Gap Analysis):** Система показывает, каких навыков не хватает сотруднику для его текущей или целевой должности.
*   **Рекомендательная система:** Автоматическая подборка курсов для закрытия пробелов в навыках.

### 2.9. Модуль социального обучения и коммуникаций
*   **Форум/Сообщества:** Тематические группы по интересам, проектам, отделам.
*   **Блоги и статьи:** Возможность для сотрудников и экспертов делиться знаниями.
*   **Система комментариев и лайков** под материалами.
*   **База знаний (Wiki):** Накопление коллективного опыта.

### 2.10. Модуль уведомлений
*   **Внутренние уведомления:** Системный "колокольчик".
*   **Email-рассылки:** Напоминания, приглашения, новости.
*   **Push-уведомления** (при наличии мобильного приложения).

---

## 3. Нефункциональные требования

*   **Производительность:** Стабильная работа при одновременной работе сотен/тысяч пользователей.
*   **Масштабируемость:** Возможность наращивать мощности с ростом компании.
*   **Безопасность:** Защита персональных данных, шифрование передаваемой информации, регулярное резервное копирование.
